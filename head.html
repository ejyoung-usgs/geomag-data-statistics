Geomagnetism Real-Time Data Statistics

How to use the data

At the beginning of each day, the download page for Automated Downloads receives a 1-day file containing dead data values (99999.00) for each minute or second of the day. As new data arrives, it over-writes the dead values in the appropriate row of the file. This means that it is necessary to re-download the same file repeatedly in a single day, and if you're looking for results nearing 100% complete data for continuous, near real-time, time-series data you should be grabbing the data from the row in the file that is 10 minutes behind current time.

Why the data is delayed

If you're pulling near-realtime geomagnetic data at regular intervals, it may help to understand the inherent delay between the time that data are acquired and the time that data become available on the website. The delay includes data transmission from an observatory to a central distribution system and transmission from the distribution system to the webpage. At present, the observatories transmit 1-minute data in 5-minute blocks, yielding a maximum delay of 5 minutes before data arrive at the distribution system. From the distribution system, data are synchronized to the webpage at 5-minute intervals, adding an additional delay of up to 5 minutes. This means that 1-minute data will typically be delayed by up to 10 minutes from the time of acquisition to availability on the webpage.

What is in the tables

These tables show the average amount of data that you will be able to access/use over the total data that should be available based on the delay from actual real-time that you use. For example, if a Delay of 10 minutes is selected and the H value for BOU reads 98.33%, this means that 98.33% of Boulder's H data arrived within 10 minutes of the moment that it was collected since this page began watching the data (indicated by the uptime), or 59 out of the 60 minutes of an hour had values while 1 value hadn't arrived yet. These tables are updated approximately every 15 minutes.
