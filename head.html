Geomagnetism Real-Time Data Statistics

1 Minute data

If you're using a tool like wget to automatically pull real-time geomagnetic data it may help to understand how the data works. There is a time delay from the moment that the data is collected until it is available on the website just by the nature of the systems. Our systems can't handle sending every second of data as it is collected, yet. So they are packaged into larger chunks (normally 5 minutes) for delivery from the observatory to our distribution system. From there we have another delay as any new data that has arrived to the distribution system is written to a file and rsynced to the web page every 5 minutes. This means that the data will normally be delayed by about 10 minutes from the moment it is collect until it can be presented.

At the beginning of each day, a new file is created with dead values (99999.00) for each of H, E, Z and F for every time stamp for that day. When new data comes in it over-writes the appropriate row in that file with actual data.
